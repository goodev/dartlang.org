Web UI Feature Summary

Web UI is deprecated. Instead, use Polymer.dart, which supersedes Web UI beginning with version 0.5 and provides many fixes and improvements. We encourage Web UI users to upgrade to Polymer.dart. The information on this page applies only to Web UI.

Written by Sigmund Cherem
November 2012 (Updated December 2012)

The Web UI package (Web UI for short) provides web components and templates to help you write web applications at scale. The following tables collect all features supported by Web UI. Read the reference specification for more details, or refer to our explainer article for a high-level introduction and examples.

Web components features

Feature Syntax example Purpose
Component tag name declaration 
<element name='x-foo-bar' ...>
 Names a component, which lets you instantiate it as <x-foo-bar>. If the constructor attribute is not set, the component behavior is defined by the class FooBar. Names must start with the x- prefix.
Component constructor declaration 
<element ... constructor='MyClass'>
 Associates the Dart class MyClass with the component's behavior. If omitted, the name of the Dart class is derived from the name of the element (see tag name declaration, above).
Tag extension 
<element ... extends='span'>
 Specifies the base element that is being extended by a component. The base element can be an HTML element or another component.
Instantiation 
... <x-tag></x-tag>
... <span is='x-tag'></span>
 Lets you use a component in your page or within another component. You can instantiate a component in two ways: using the tag directly (<x-tag>) or using the is attribute on the base HTML element that the component directly or indirectly extends from (<span is='x-tag'>).

Note: Do not omit the closing tag.
Retrieval 
document.querySelector('x-foo').xtag
 Retrieves a reference to a component previously instantiated in your page.

Note: The xtag fields will be null within main, but will be available at the end of the event loop.
Element's template 
<element name='x-foo'>
  <template> ... </template>
 Specifies the markup of the component. The template portion of a web component is inert until the component is instantiated. The template may contain any of the templating features (data bindings, conditionals, loops, and so on) and also the content redistribution available in web components.
Element's script 
<element name='x-foo'>
  ...
  <script type='application/dart'>...</script>
 Specifies the behavior of a component. This script defines the class corresponding to the component.
Element's style 
<element name='x-foo'>
  ...
  <style> ... </style>
 Specifies scoped CSS rules that are only applicable in the context of the component's <template> body.
Content insertion 
<element name='x-foo'>
  <template>
    ...
    <content></content>
 Allows components to have children. When a component has children, those children go where the <content> tags are. For example, consider this usage: 
<x-foo><div>hello</div></x-foo>
The <div>hello</div> will be placed within the element's template where the content tag is specified.
Selector in content insertion 
<element name='x-foo'>
  <template>
    ...
    <content select='div'></content>
    <content></content>
 Specifies which subset of a component's children are distributed in a particular content tag. For example, consider this usage: 
<x-foo><span>one</span><div>two</div></x-foo>
The <div>two</div> will be placed at the first <content> insertion point (where div tags are selected), and the <span> will be added at the insertion point of the second content tag.
Base component insertion 
<element ... >
  <template>
    ...
    <shadow></shadow>
 Embeds the content of a base component. When a component extends another component, The <shadow> tag is an insertion point where the contents of the parent component are added.
Components inclusion 
<html ...>
  <head>
  <link rel="import" href="...">
  </head>
 Imports component definitions from the URL specified by href. You can use these components within the body of the current HTML page and within components that the current HTML page defines.

Templating features

Feature Syntax example Purpose
Text node data binding 
<div>{{exp}}</div>
 Injects the value of evaluating exp in the document and watches for changes. Any time a change to exp is detected, the UI is updated. Values of the special type SafeHtml (from package:web_components/safe_html.dart) are treated in a special manner. If the value of exp is not SafeHtml, the contents are converted to a string and are treated as text (they are escaped as safe HTML). Otherwise, if the value of exp is a SafeHtml, the contents are injected directly as an HTML fragment.
Attribute data binding 
<td colspan="{{exp}}"></td>
 Binds the value of exp to the value of the attribute. Similar to binding in text nodes, the expression is watched for changes and the element attribute is updated accordingly. The type of the expression is treated accordingly, so unsafe content is appropriately escaped as an attribute value.
Class attribute data binding 
<div class="{{Class1}} {{Class2}}"></div>
<div class="{{classesAsList}}"></div>
<div class="{{classesAsString}}"></div>
 Binds the values to a class attribute. Similar to attribute bindings, except that the system is smart enough to update the class list by adding and removing individual classes that change. If the expression is null, it means that the class(es) represented by that expression are all removed.
Style attribute data binding 
<div style="{{exp}}"></div>
 Binds the value of exp to the value of a style attribute. Similar to attribute bindings, except that exp is expected to be a map and the system updates the style of the node by treating the key-value pairs in exp as CSS property-value pairs.
Two-way data binding in interactive elements 
<input type="text"
       bind-value="assignableValue">
<textarea bind-value="assignableValue">
</textarea>
<input type="checkbox"
       bind-checked="assignableValue">
 Directly updates assignableValue with user input. Like data bindings in attributes, this displays the latest value of the assignableValue expression in the element. Additionally, when the element is updated due to UI interaction, the assignableValue is also updated and kept in sync.
Conditional template node 
<template instantiate="if exp">
contents
</template>
<template if="exp">contents</template>
 Conditionally adds contents if exp evaluates to true. The contents are added after the template tag (not as children of the template tag). To make a conditional row or cell in a table, use conditional tag attributes instead.

Note: both instance="if exp" and if="exp" are accepted, the syntax is not yet finalized.
Conditional tag attribute 
<div template instantiate="if exp">
contents
</div>
<div template if="exp">contents</div>
<table><tbody>
<tr>
  <td template instantiate="if showCell">
    contents
  </td>
</tr>
</tbody></table>
 Conditionally adds the element to the DOM tree if the expression is true. This can be used to make rows and cells conditionally visible.

Note: both instance="if exp" and if="exp" are accepted, the syntax is not yet finalized.
Iterating template node 
<template iterate="x in exp">
contents
</template>
 Inserts contents for each item in exp (an iterable collection). The contents are added after the template tag (not as children of the tag). To iterate over table rows or cells, use iterating tag attributes instead.
Iterating tag attribute 
<table>
<tbody template iterate="x in exp">
  <tr>
    <td>{{x.firstName}}</td>
    <td>{{x.lastName}}</td>
  </tr>
</tbody>
</table>
 Expands contents under the element, once per item in the collection that exp evaluates to. This can be used to create rows and cells in tables.
Inline event listener 
<div on-click="myHandler($event)">
contents
</div>
 Binds a UI event to a Dart expression. Whenever the event fires, the associated expression is executed and watchers are notified about possible changes. The fired event is stored in the special variable $event, available in the scope of the Dart expression.
