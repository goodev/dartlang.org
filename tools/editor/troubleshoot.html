解决 Dart 编辑器遇到的问题

了解你的系统架构

如果你不知道你的系统是 32-bit 还是 64-bit 的 Linux， 使用命令 uname -a。 如果输出的内容包含 64，则你是 64-bit Linux； 否则你就是 32-bit Linux。

几乎所有的 Mac OS X 系统都是 64-bit 的， 所有我们推荐你用 64-bit Dart 编辑器。 如果你遇到了问题， 则尝试下 32-bit 版本。

如果你不知道你是 32-bit 还是 64-bit Windows，参考 Microsoft 的支持站点。

启动 Dart 编辑器

你的 Java 应该和 Dart 编辑器用一样 的 bit 版本。 例如 如果你用 64-bit 的 Dart 编辑器， 则应该安装 64-bit 的 Java。 否则的话，当启动 Dart 编辑器的时候 可能会看到类似如下的一个错误消息： 

Failed to load the JNI shared library "C:\Program Files(x86)\Java\jre6\\bin\client\jvm.dll

用如下命令，可以查看已经安装 Java 的 bit 信息 java -version。

如果在 64-bit Linux 上用 32-bit Java 则 Dart 编辑器 不能使用。

解决该问题的方法： 用 64-bit Java JDK。 可以从 Java SE Development Kit 7 下载页面 下载 64-bit JDK 7 。 如果已经安装了 64-bit JDK ，但是默认用的不是，则你可以 指定 Java 运行环境。

当在 Mac 上启动 Dart 编辑器可能会遇到一些问题：

  • Unidentified developer
  • "DartEditor.app" is damaged (64-bit only)
  • Java installation failure
  • Slow startup
  • 32-bit Mac

Unidentified developer

如果你的系统是 Mac OS 10.8 (Mountain Lion)， 由于 Dart 编辑器 不是从 Mac App 商店安装也不是认真过的开发者， 所以你的电脑可能不能启动 Dart 编辑器。 在我们解决该 (bug #5088)之前，你 可以自己按照如下方式解决。

最简单的方式就是在第一次启动 Dart 编辑器 的时候 不要双击 Dart 编辑器 图标。 而是用右键点击（或者 Ctl+click ），在弹出的菜单中选择 Open。 然后在弹出的对话框中选择 Open。

另外一种方式就是修改安全设置。 在 System Preferences > Security & Privacy 设置界面中。 在 General tab 页的 Allow applications downloaded from: 中， 选择 Anywhere。 现在应该可以启动 Dart 编辑器了。 启动 Dart 编辑器后， 你可以把安全设置再修改回去。

"DartEditor.app" is damaged (64-bit only)

有时候当第一次启动 64-bit Dart 编辑器时， Mac OS X 会错误的显示如下信息：

"DartEditor.app" is damaged and can't be opened. You should move it to the Trash.

解决方式是下载并安装 32-bit版本的 Dart 编辑器。

Java installation failure

当没有 Java SE 6 时安装 Dart 编辑器 ， Dart 编辑器 将为你安装 Java。 如果无法安装，你可以重新 启动 Dart 编辑器 或者 自己安装 Java 。

Slow startup

如果你使用了 Sophos Anti-Virus， Dart 编辑器 (已经其他 Java 应用) 比其他应用启动时间要多 10-15 倍。

解决方式：关闭扫描 JAR 文件的选项。 例如：

  1. 在 Sophos Anti-Virus 设置界面中， 选择 On-access Scanning。
  2. 如果设置界面锁定了， 点击锁图标并输入你的密码。
  3. 点击 Excluded Items 然后点击 + 符号。
  4. 找到包含 DartEditor 的目录。
  5. 搜索该目录下依 .jar 结尾 的文件。
  6. 选择所有的 .jar 文件，并 点击 Open。

32-bit Mac

如果你在使用非常罕见的 32-bit Mac， 你不能运行 64-bit 版本的 Dart 编辑器。 将会出现如下警告：

解决方式：下载并安装 32-bit 版本的 Dart 编辑器。

指定 Java 运行环境

如果有必要你可以设置 Dart 编辑器 需要的 Java 环境。 在 Dart 安装目录中找到 DartEditor.ini 文件， 在 ‑vmargs 代码之前添加 如下代码： 

-vm
/full/path/to/java

重要： 在 Windows 系统中，用双反斜杠 (\\) 作为目录分割符号： 

-vm
c:\\bin\\java_jdk1.7.0\\bin\\javaw.exe

例如，修改后的 DartEditor.ini 类似如下内容： 

...
-vm
/usr/local/buildtools/java/jdk-64/bin/java
-vmargs
...

启动 Dartium

默认情况下，当点击 Dart 编辑器的 Run 按钮 ， 你的应用会在 Dartium 中运行。 如果你看到一条信息告诉你需要 下载新版本的 Dartium， 则说明你的 Dartium 过期了。

解决方式： 下载 并安装新版本的 Dart 编辑器。 里面包含一个升级过的 Dartium，和 最新的 Dart 编辑器。 或者 单独下载一个新版本的 Dartium 并把该文件复制你 Dart 安装目录中。

使用 Eclipse

如果 Dart 编辑器 和 Eclipse 使用同一个 workspace 目录， 他们会保存冲突的配置文件， 这样在 Dart 编辑器 和 Eclipse 中运行都会出错。

解决方式：不要把 Dart 编辑器 和 其他 Eclipse 软件使用同一个 workspace。

清除所有设置

Dart 编辑器 的设置文件保存在平台目录中， 所有当更新 Dart 编辑器的时候不会丢失配置信息。 如果你的设置信息出问题了， 或者你想重头开始， 则可以删除设置目录。

下面显示了不同平台上 Dart 编辑器 设置文件保存的位置。

平台 设置位置
Windows    <user's home directory>\DartEditor
Linux $HOME/.dartEditor
Mac $HOME/Library/Application Support/DartEditor
