开发者指南

Dart 开发概览告诉你一些能够 帮助你创建、测试和部署 Dart 代码的 文档、文章和其他开发资源。

开始

阅读下面内容， 或者开始下载 Dart 并玩玩 Dart 编辑器。

海盗们 停下来：编写个 Web 应用吧 ：一步一步创建一个 web 应用 。 这个一小时代码实验室假设你具有其他编程语言背景， 但是并不要求对 Dart 或者 HTML 有所了解。

语言概览 (Dart: Up and Running, Chapter 2) ： 介绍 Dart 语言中的每个主要功能特性， 从变量到操作符、从类到库。

Dart 教程 ：使用 Dart 创建 web 应用的手把手教程。

概念

目前你不需要查看下面的所有内容， 但是要记得这些内容以方便你以后参考。

库

下载的 Dart SDK 中包含了一些库。 你也可以定义自己的库或者 使用pub 包管理器 来获取其他的库。

库和可见性 ( Dart: Up and Running Chapter 2 的一部分) ： 学习如何使用和创建库。

库概览 (Dart: Up and Running, Chapter 3) ：学习如何使用 每个 Dart 库中的主要特性。

文章 ：从”库和 APIs”部分学习如何 使用独立的库、包和 API，例如 polymer.dart、 单元测试、 和 Futures。

API 文档 ：查看 Dart 库的所有 API 文档。

核心基础类

很多 Dart 函数返回 Iterable、Future或者 Stream 对象。

Iterable

一个 Iterable 对象是一个 对象的集合，例如 list 或者 set。 Dart 语言中的 Iterable 类提供了 用于 for-in 循环的接口。 深入了解：

  • Collections ( Dart: Up and Running Chapter 3 的一部分)
  • Iteration ( Dart: Up and Running, Chapter 3 的一部分)
  • Iterable API 文档

Future

Future 对象代表一个未来时刻获取到的值。 例如， dart:io 中的 File 类的 readAsString() 函数 返回一个 Future<String>， 你可以使用她获取整个文件的内容为一个字符串。 深入了解：

  • 使用 Future APIs
  • Futures 和 错误处理
  • Future API 文档

Stream

Stream 对象代表一个未来时刻获取到的 一系列数据。 Stream 的一个示例是 dart:html 中的按钮的 onClick 事件流。 另外一个示例就是 dart:io 的 File 类的 openRead()函数的返回值， 用来读取文件数据流。 深入了解：

  • 用 Streams 数据
  • 在 Dart 中创建 Streams
  • Stream API 文档

可以用 Dart 干任何事情，从脚本到 web 服务器。 感谢 Heroku 支持第三方运行环境， 你可以在 Heroku 云服务器中运行 Dart 应用的服务器端代码。

创建 Web 应用

如果你对 web 应用还不了解， 请先看看 Dart 教程— 假设你没有 Dart 和 web 应用的开发经验， 通过一些列教程告诉你如何用 Dart 编写 web 应用，

当你开发 web 应用的时候， 可以在 Dartium 中运行。 开发完成后，可以把你的 Dart 代码编译 为 JavaScript 并在其他浏览器中运行。

当创建 web 应用的时候，你可以用 dart:html 和 polymer.dart 包 或者 AngularDart。 虽然你可能不需要用 polymer.dart 或者 AngularDart， 但是这些工具简化了你的代码。

dart:html

工具

Dart 工具非常友好，你 可以选择不同的编辑器、IDE以及其他工具。

当 下载 Dart后， 只包含Dart 编辑器 IDE、 Dartium (带有 Dart 虚拟机的浏览器) 和 Dart SDK。 SDK 包含一些命令行工具，分别为： Dart VM (dart)、一个 Dart-to-JavaScript 编译器 (dart2js)、 和一个包管理器 (pub)。

Dart 编辑器 ：开始学习 Dart 的最简单方式。

Dart 工具 ：关于其他工具的一些信息和 下载地址。

测试

  • 为 Pub 包编写单元测试
  • Dart 中的单元测试
