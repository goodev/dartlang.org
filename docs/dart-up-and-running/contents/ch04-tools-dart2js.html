dart2js: Dart-to-JavaScript 编译器

可以用 dart2js 工具把 Dart 代码 编译为 JavaScript。 Dart 编辑器 就是用 dart2js 把代码编译为 JavaScript 并在其他浏览器中运行的。 本节内容介绍如何用 dart2js 命令行工具。 同时也介绍了如果调试 dart2js 编译的 JavaScript 程序。

基本用法

下面是把 Dart 编译为 JavaScript 的示例：

lang-sh
dart2js --out=test.js test.dart

上面的命令把 Dart 代码编译为功能一样的 JavaScript 代码。 同时还生成了一个代码映射表，这样可以方便的调试生成的 JavaScript 代码。

支持的参数

dart2js 常用的参数有：

-o<file> 或者 --out=<file>

编译后的文件保存在 <file> 文件中。如果 没有设置该参数，生成的 JavaScript 文件名为 out.js.

-c 或者 --checked

插入运行时类型检测和启用断言（检测模式）。

--minify

生成最小尺寸的代码（也就是启用代码压缩）。

-h 或者 --help

显示帮助信息。(使用 -vh 显示所有的帮助信息)

--output-type=dart

输出的文件为 Dart 代码而不是 JavaScript。 在部署代码的时候该选项非常有用， 该选项会生成一个文件包含该应用的所有 Dart 代码，这样可以加速 web 页面的加载。

帮助 dart2js 生成更好的代码

你可以通过如下方面来帮助 dart2js 生成更好的代码：

  • 编写能够很容易推断类型的代码。

  • 当部署代码的时候， 用 dart2js --minify 参数来减少生成的代码。

Note

不用担心你使用的各种库的大小。 使用 tree shaking 技术， dart2js 会忽略 没有用到的类、函数、方法、等等。你只要导入需要使用的库就可以了， 让 dart2js 帮助你删除不需要的代码。

如下是一些帮助 dart2js 生成更好代码的最佳实践，遵守这些实践可以 让 dart2js 生成更简单、运行更快的 JavaScript 代码：

  • 避免设置变量值为 null。

  • 如果可能总是使用 const 或者 final 修饰符。

  • 使用一致的类型参数来调用同一个函数或者方法。

调试

下面是一些在 Chrome、 Firefox、 和 Safari 中调试 dart2js 生成的 JavaScript 代码的一些建议。 在支持代码映射的浏览器中调试 dart2js 生成的 JavaScript 代码是非常容易的， 例如 Chrome。

无论使用那个浏览器，你至少要启用在非捕获异常发生时暂停执行的功能，如果启用 所有异常则更好。 对于 dart:isolate 和 dart:async 这些框架来说，用户代码在 try-catch 中执行， 所有我们建议启用所有异常发生都暂停执行的功能。

Chrome

在 Chrome 中调试：

  1. 打开 Developer Tools 窗口。具体如何打开，请参考 How to Access the Developer Tools。

  2. 启用代码映射功能。详细信息参考这个视频 SourceMaps in Chrome。

  3. 启用调试功能，选择所有异常或者非捕获异常。详细信息参考 Pause on Exceptions 和 Pause on Uncaught Exceptions 文档。

  4. 重新载入调试的应用。

Firefox

Firefox 还不支持代码映射 (参考 bug #771597)。 在 Firefox 中调试步骤：

  1. 启用 Developer Toolbar。启用方式参考 Kevin Dangoor 的文章 New Firefox Command Line Helps You Develop Faster.

  2. 点击浏览器底部的 Debugger 按钮 ，并启用 Pause on exceptions。(参考 Figure 4.7, “Firefox’s Developer Toolbar”.)

  3. 重新载入调试的应用。

Figure 4.7. Firefox’s Developer Toolbar


Safari

在 Safari 中调试的步骤：

  1. 打开 Develop 菜单。详细操作参考 Safari Web Inspector Guide.

  2. 启用断点。参考 Safari Debugging 页面的 Figure 4-2 图。

  3. 重新载入调试的应用。
