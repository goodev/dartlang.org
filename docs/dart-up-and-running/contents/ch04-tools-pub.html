pub: Dart 包管理器

你可以用 pub 来 管理 Dart 包。一个 Dart 包只是一个包含多个 Dart 库和其他资源的目录。 包还可以包含文档、测试、图片等资源。 应用使用其他包的前提条件是该应用 需要为一个包。

Note

Dart 编辑器支持 pub 的功能，包含 创建、安装 、更新、以及发布包等。

包可以位于任何地方。例如，一些包可能在 GitHub 上。 Dart 团队在 pub.dartlang.org 发布 包，我们也希望你在这里发布你的包。

要使用 Dart 包中的库，你需要安装如下 步骤操作：

  1. 创建一个 pubspec (该文件列出了 库的依赖关系和其他元数据，例如 你的包 的名字 等)。

  2. 用 pub 获取你的应用所依赖的包。

  3. 导入要使用的库。

创建 Pubspec

要使用其他包，你的应用必需创建一个 pubspec 文件并列出 所依赖的包和包的下载地址。 pubspec 文件的名字为 pubspec.yaml，该文件必需 位于你项目的根目录中。

下面是一个示例 pubspec 文件，指定了两个包。 第一个使用了位于 pub.dartlang.org 的 js 包； 第二个使用了位于 Dart SDK 的 intl 包： 

lang-js
name: my_app
dependencies:
  js: any
  intl: any

详情请参考：pubspec 文档 ，这里还包含了你可以使用的第三方包的介绍。

安装包

定义好 pubspec 文件后，就可以在项目根目录中运行 pub get 来下载这些包：

lang-sh
cd my/app
pub get

该命令检测你的应用所依赖的包，并把这些包下载到一个中央缓存目录。 对于 git 包，pub 则会 clone 该库的 git 库。而对于托管到文件服务器的库， pub 从 pub.dartlang.org 下载。 同时还下载了应用使用的包所依赖的包。 例如，js 包依赖 unittest 包， 则 pub 会同时下载 js 和 unittest 包。

最后，pub 在你的应用根目录下创建一个 packages 目录，该目录连接了你应用中需要使用的包。

从包中导入库

要使用包中的库，使用 package: 前缀导入库：

lang-dart
import 'package:js/js.dart' as js;
import 'package:intl/intl.dart';

Dart 运行环境将负责解析package: 并从 packages 目录中加载需要的库。

其他信息

运行 pub --help 来查询其他命令。关于 pub 的其他信息以及 如何发布你的包，请参考 pub 文档。
