Dart 编辑器

在 the section called “开始使用 Dart” 我们已经介绍了 Dart 编辑器了。 下面是使用 Dart 编辑器的一些有用的提示，例如 如何指定浏览器 和 编译为 JavaScript。 如果你遇到了问题，请参考 Dart 编辑器问题指南。 Dart 编辑器更新比较快， 所以这里的截图可能看起来不太一样。 最新的信息 ，请参考 Dart 编辑器文档。

查看示例项目

Dart 编辑器的欢迎界面 (Figure 1.2, “Dart 编辑器和欢迎界面”) 显示了一些示例项目。 只需要点击示例项目图片就可以打开该项目并查看其 代码了。

如果你没有看到欢迎界面，很有可能是之前你关闭该页面了。 通过菜单 Tools > Welcome Page 可以打开该页面。

管理文件视图

文件视图显示了 库的文件、安装的包的文件、和你的应用 创建的文件。

添加应用

下面是如何打开应用并在文件 视图中显示的步骤：

  1. 打开 File 菜单，然后选择 Open Existing Folder... ， 或者用快捷键 (Ctrl+O 、在 Mac 系统用 Cmd+O)。

  2. 找到 包含应用的目录。

  3. 点击 Open。

在文件视图中就可以看到该目录和其中的文件了。

删除应用

你可以从文件视图中删除应用，可以同时删除磁盘上的文件也可以保留 文件。

右键 (或者 Ctrl+点击) 项目目录并选择 Delete。 如果你想同时删除该项目在磁盘上的文件，则在删除对话框中选中 Delete project contents on disk。

创建应用

从头创建一个命令行应用或者 web 应用也是很简单的：

  1. 点击 New Application 按钮 (在 Dart 编辑器的左上角)。 或者选择菜单 File > New Application 。然后出现一个对话框。

  2. 输入应用的名字，比如 hello_web。如果你不想用默认的 目录存放该项目，你可以选择一个新的目录。

  3. 如果你真的不想让编辑器创建一些示例内容，则不要选中 Generate sample content 。

  4. 选择要创建的应用类型，例如 命令行应用、 web 或者使用 Polymer.dart 的 web 应用。

  5. 点击 Finish 来 创建一个带有一些示例文件的项目。

    在编辑视图中会打开默认的 Dart 文件，在 文件视图中会显示项目目录。 创建后的 Dart 编辑器看起来应该和下图类似： Figure 4.1, “使用默认示例代码创建的新应用”。

    Figure 4.1. 使用默认示例代码创建的新应用


编辑应用

Dart 编辑器提供了一些编辑功能， 例如 Dart 代码辅助、API 浏览、支持重构以及搜索文件等。

使用代码辅助

代码辅助功能看起来如 Figure 4.2, “代码辅助建议” 所示。当你输入如下代码的时候，提示出现：

  • 输入一个类或者变量名字，然后再输入一个圆点。

    例如 document. 或者 DateTime. 然后暂停一小会。 当代码建议面板出现后，在继续输入可以过滤建议的代码。

  • 按 Ctrl+Space。

    例如，输入 Dat 然后按 Ctrl+Space 来查看 以 “Dat” 开头的类。

当代码建议面板出现后，你可以点击、输入或者用上下箭头来选择你先要的建议。 按回车键（Enter）或者返回键（Return）选择代码建议，用 退出键（Esc） 来关闭代码建议面板。

Figure 4.2. 代码辅助建议


浏览 APIs

使用 Dart 编辑器可以很容易的查找 API 是在何处定义的。 同时还可以列出所选 Dart 文件的内容大纲视图。

查看 API 的更多信息

要查看一个 API 的详细信息 -变量、函数、类型、 库、等等 - 只需要在 编辑视图上把鼠标放到 API 名字上即可。 Dart 编辑器会弹出一个面板来显示关于该 API 的更多信息。

不管该 API 是否在当前文件定义，都可以通过如下方式找到该 API 的定义位置：

  1. 在 Dart 文件的编辑视图中，点击一个 API 。 Dart 编辑器会高亮当前文件 所有使用该 API 的地方。

  2. 然后在该 API 上右键。

  3. 选择菜单中的 Open Declaration 选项。

    编辑器会打开该 API 的定义位置。 例如，如果你查询 querySelector 的定义位置，则编辑器 会打开定义 querySelector() 方法的文件。

浏览文件大概

通过快捷键 Alt+O (Mac 上为 Option+O ) 或者 右键并选择 Outline File 菜单。

然后会弹出一个面板，在该面板中显示了当前文件定义的 类、函数、和变量。 例如， Sunflower 示例中的 sunflower.dart 文件的大纲 面板如图所示： Figure 4.3, “Sunflower 示例的大纲面板”

Figure 4.3. Sunflower 示例的大纲面板


你可以输入字符来过滤显示的内容。例如 如果你输入 c，则只会显示 center 和 context 变量。

如果你选择了其中的一个条目，例如 centerX— 编辑器会把文件滚动到该条目定义的地方显示。

另外，还有一个常驻的 大纲视图，通过菜单 Tools > Outline来打开。

重构

要修改代码的名字，在编辑文件视图中，可以把鼠标放到名字中间或者双击名字，然后右键选择菜单 中的 Rename...选项。

你几乎可以重命名任意内容，局部变量、方法参数、成员变量、函数、类型、顶级方法、库前缀 、等等。

搜索

Dart 编辑器右上角的搜索输入框可以 直接定位到 如下位置：

  • 类型定义

  • 文件

  • 文件中的文本

搜索的范围为当前文件视图中的所有文件。 在搜索视图中显示搜索结果。 在搜索视图中，双击一个文件可以在编辑视图中显示该文件。 在打开的文件中， 所有符合搜索的字符串都被高亮显示。

运行应用

只需要选择该应用的文件并点击 Dart 编辑器的 Run 按钮 就可以运行该应用。 如果是 web 应用， Dart 编辑器会启动一个浏览器来运行该应用。

当用 Dart 编辑器运行 web 应用的时候，默认 Dart 编辑器会启动 带有 Dart VM 的 Dartium 浏览器来直接运行 Dart 代码。 如果你指定了其他的浏览器，则 Dart 编辑器用 dart2js 来把 Dart 代码 编译为 JavaScript 在浏览器中运行。

设置启动配置

通过菜单 Run > Manage Launches 来设置多个启动配置。

对于 Web 应用，你可以设置如下内容：

  • 要打开的 HTML 文件或者 URL 地址

  • 传递给浏览器的参数。例如， --allow-file-access-from-files

  • 检测模式(checked mode) (只支持 Dartium )

  • 启用实验性功能 (只支持 Dartium)

  • 在哪里显示浏览器的 stdout 和 stderr 输出内容 (只支持 Dartium ，可以帮助诊断 Dartium 崩溃问题)

例如，一个 web 应用可以有一个启动 Dartium 的配置和启动其他浏览器的配置。

对于命令行应用，可以设置如下内容：

  • 执行的 .dart

  • 工作目录

  • 传递给应用的参数

  • 检测模式

在生产模式下运行

默认情况下，应用在 检测模式 运行.如果要在生产模式下运行，则可以在启动 配置中禁用检测模式：

  1. 最少要运行一次该应用，这样就生成该启动配置项了。

  2. 选择 Run > Manage Launches或者点击 Run 按钮 Run button 右边的三角形图标并选择 Manage Launches。

  3. 在 Manage Launches 对话框中，找到你应用的启动配置。 如果已经存在了，则选中该配置。

  4. 不要选中 Run in checked mode (参考 Figure 4.4, “要在生产模式运行，不要选中 checked mode”)。

    Figure 4.4. 要在生产模式运行，不要选中 checked mode


  5. 点击 Apply 来保存 修改，或者点击 Run 来保存并 运行应用。

关于检测模式和生产模式的更多信息，请参考 the section called “运行时模式”。

设置运行 JavaScript 的浏览器

要设置一个浏览器来运行你应用的 JavaScript 版本，安装如下步骤操作：

  1. 选择 Dart Editor > Preferences.

  2. 点击 Run and Debug.

  3. 如果你不想用系统默认浏览器，则不要选中 Use default system browser ,并 设置你想使用的浏览器文件位置。 例如： /Applications/Firefox.app

  4. 点击 OK 保存 修改。

现在 只要你用 JavaScript 来运行你的应用， Dart 编辑器都会打开你选择的浏览器。

你也可以用默认的配置打开 JavaScript 应用。然后把网址复制到你想测试的浏览器中即可。

调试应用

在 Dart 编辑器中可以调试命令行应用和 web 应用。下面是一些提示：

  • 在编辑视图代码每行左边距区域双击可以添加断点。

  • 用 调试器视图（Debugger view）来查看应用的状态和控制代码的执行。 默认情况下，调试器视图在编辑视图的右边，当你第一次执行到断点代码位置处时，调试器视图自动打开 (Figure 4.5, “用 Dart 编辑器调试 Sunflower 示例应用”).

  • 要查看变量的只，把鼠标放到变量名字上，在 调试器视图 查看，或者用 Expression （表达式） 查看.

  • 要调试 web 应用，需要用 Dartium 配合 Dart 编辑器。 当调试的时候， 编辑器接管 Dartium 控制台。例如， 编辑器可以显示 print() 语句打印的消息。

Figure 4.5. 用 Dart 编辑器调试 Sunflower 示例应用


编译为 JavaScript

要把你的代码编译为 JavaScript 你无需做额外的工作。 当你在启动配置中 设置了其他浏览器，则 Dart 编辑器 会在执行应用之前自动把 Dart 代码编译为 JavaScript。

当然，你也可以不允许应用就可以把 Dart 代码编译为 JavaScript。 通过菜单 Tools > Generate JavaScript即可。另外还可以用命令行工具 dart2js (参考 the section called “dart2js: Dart-to-JavaScript 编译器”).

其他功能

Dart 编辑器还有很多功能，例如 生成文档、自定义界面和快捷键等。

用 dartdoc 生成文档

用工具 Tools > Generate Dartdoc 菜单来生成 Dart 代码的 HTML 文档。 关于文档格式的更多信息，请参考 the section called “文档注释”.

自定义 Dart 编辑器

在 Preferences 对话框中你可以自定义编辑器的字体、边距、快捷键和其他内容。通过菜单 Tools > Preferences ( Mac: Dart Editor > Preferences) 来打开该对话框。

你还可以定制显示那些视图，和视图显示的位置和大小。通过 Tools 菜单添加视图。通过视图的 X 图标来删除视图。 拖动视图 tab 来移动视图到另外一个位置，可以移动到 Dart 编辑器的其他地方，也可以移出 Dart 编辑器。 拖动视图边缘可以放到缩小。

快捷键提示

通过菜单 Help > Key Assist (Figure 4.6, “Help > Key Assist”)可以查看可以使用的快捷键。
