Dartium: 带有 Dart VM 的 Chromium

本节内容告诉你如何下载和使用 Dartium 浏览器。 Dartium 直接运行 Dart web 应用中的 Dart 代码， 这样你无需把 Dart 代码编译为 JavaScript 就可以 测试你的应用了。

Warning

该浏览器目前还处于开发早期。 请不要用 Dartium 作为你的日常使用浏览器。

下载安装 Dartium

如果你已经下载了最新的 Dart 编辑器，则里面已经包含了 Dartium.

如果你还没有下载 Dart 编辑器或者你想下载一个其他版本的 Dartium ，你可以从 下载页面下载。 Dartium 程序在一段时间以后就会失效。 所以当你的 Dartium 失效后，你需要从新下载一个。

只需要解压下载的文件 就安装好 Dartium 了。 如果你想让 Dart 编辑器启动你下载的 Dartium，则需要把下载的文件复制到 dart-sdk 目录中的 Dartium 目录下 (参考 the section called “第一步：下载并安装 Dart”) ，并替换之前下载的 Chromium。

启动 Dartium

要启动 Dartium， 只需要双击 Chromium 可执行文件即可。 或者按照 the section called “运行应用” 描述的方式在 Dart 编辑器中启动，或者按照 the section called “从命令行启动 Dartium” 描述的方式在命令行中启动。

Warning

已经使用 Chromium 的用户请注意： 如果另外一个 Chromium 在运行，当运行 Dartium 的时候可能会出现配置冲突的情况。 通过命令行 --user-data-dir 参数来启动 Dartium 可以避免该问题。

提交 Bug

如果你发现了一个 Dartium 的 bug，可以按照 Dartium bug 模板 来给 Dart 开发组提交 Bug。

连接到 Dart 代码

用类型为 application/dart 的 script 来连接 Dart 代码。 例如：

lang-html
<!DOCTYPE html>
<html>
  <body>
    <script type="application/dart" src="app.dart"></script>

    <!-- Support for non-Dart browsers. -->
    <script src="packages/browser/dart.js"></script>
  </body>
</html>

Note

Dart 编辑器自动在项目的启动 HTML 文件 中添加 application/dart script 和 dart.js script 标签。

检测浏览器是否支持 Dart

可以通过 JavaScript 代码检测浏览器是否支持 Dart： 

lang-js
// In JavaScript code:
if (navigator.userAgent.indexOf('(Dart)') == -1) {
  // No native Dart support...
  window.addEventListener("DOMContentLoaded", function (e) {
      // ...Fall back to compiled JS...
    }
  }, false);
}

从命令行启动 Dartium

由于 Dartium 是基于 Chromium 的，所有 Chromium 标记( flag ) 都应该可以使用。有些情况下，你可能想使用 Dart 相关的标记来设置 Dart VM 的行为。 例如，在开发 web 应用的时候，你想让 VM 检查类型声明和断言（assertions）， 启用 检测模式 即可启用该功能 (VM 的 --checked 标记)。

在 Linux，可以通过如下方式启动 Dartium： 

lang-sh
DART_FLAGS='--checked' path/chrome

Mac 系统:

lang-sh
DART_FLAGS='--checked'              \
  path/Chromium.app/Contents/MacOS/Chromium

Mac 系统也可以这样 :

lang-sh
DART_FLAGS='--checked'              \
   open path/Chromium.app

Note

通过 chrome://version 你可以查看当前 使用的命令行标记和执行文件路径。
