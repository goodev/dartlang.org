鸣谢

你并不需要成为编程高手就能开发 Web 应用。使用 Dart ,你可以 有非常高的生产效率来为当今的互联网开 发高性能应用。

这本书的目标是成为一本实用的Dart语言入门书，包含语言介绍、类库介绍、 工具介绍。由于本书很短，而且 Dart 也很年轻，你同样需要查阅 Dart 官方网站 http://www.dartlang.org 和 Dart 中文网站 http://www.dartlang.cc —获取更多的细节以及后续更新。 关注 Google+ Dart 网页 和 Dart 中文微博来 获取最新的新闻。

另外一个重要的网站是本书的 GitHub 代码库， 本书使用 Creative Commons Attribution-Noncommercial-No Derivative Works 3.0 United States License 许可发布。这本书中例子的源代码文件在 code/ 子目录中。 直接下载源代码比从书中复制代码更加方便。

内容更新

这部分详述了第一次和第二次印刷后的所有修改。

第二次印刷后的修改

我们改变文字和例子以反应出 Dart 语言的相关修改：

  • 实例变量不能是 const 的了。 取而代之的是 static const 。

  • ? 操作符用来测试是否设置了一个可选参数的功能取消了。

  • Map 的字面量定义中的 key 值不再强制是字符串类型的了。

  • Dart 现在拥有符号（Symbol）和符号字面量 (#标识符): the section called “Symbols（符号）”。

  • 相等测试变得更容易了：the section called “判断函数相等”.

  • 提升了按位操作符的优先级。

我们也更新了以下内容以反应 API 的变化。在 Dart：核心库中：

  • Set 的 isSubsetOf() 方法不再使用了。 相取代的, 使用 Set 的 containsAll() 方法。 你可以使用 toSet() 把任何可迭代的类型转换成 Set。

  • Collection 类不再使用了, 因此我们改变了 sets，lists，和 maps 的功能，详见 the section called “集合”, 并且我们更多的关注 Iterable.

  • 先前的 dart:uri 类库中的功能现在被整合到了 dart:核心类库的 Uri 类中。 顶级函数 encodeUri(), encodeUriComponent(), decodeUri(), decodeUriComponent() 现在变成了各自对应的 Uri 类 的静态方法为： encodeFull(), encodeComponent(), decodeFull(), decodeComponent(). 要从字符串中创建 Uri，你需要使用静态的 parse() 方法。 Uri 类现在只有一个构造函数， 它依然拥有原来 Uri.fromComponents() 构造函数同样的参数。 最后, domain 域现在改名为host。

  • @deprecated, @override, 和 @proxy 注释从 meta package 转移到了 dart:core 核心类库中。

dart:io: 类库：

  • File 类的 openWrite() 方法现在不再必须要求参数了，但是仍然保留了两个可选参数。 mode 参数有一个默认的值： FileMode.WRITE。

  • File 的 readAsString() 和 readAsLines() 方法不再必须需要一个参数。 相替代的, 他们拥有了一个可选的参数： (encoding), 这个参数的默认值是 Encoding.UTF_8

  • IOSink 的写数据方法调整了。用write()取代了addString()来写字符串数据。前期临时命名为 writeBytes()的写二进制数据的方法, 改成了 add().

新的 dart:convert 类库取代了 dart:json 和 dart:uri:

  • 取消了 StringDecoder, 而使用 UTF8.decoder 转换 UTF-8 字符为 Dart 字符。

  • 取消了 LineTransformer, 现在使用 LineSplitter 对象将一个字符串输入流分拆成每一行。

  • 过去使用的 dart:json 顶级函数 parse() 和 stringify() 功能废弃了, 现在分别使用 JSON.decode() 和 JSON.encode()。

  • 过去使用的 dart:uri 顶级函数 decodeUri() 和 encodeUri() 功能废弃了, 现在分别使用 UTF8.decode() 和 UTF8.encode()。

其他的 API 改变:

  • 在 dart:html 类库中, query() 和 queryAll() 方法分别改成了 querySelector() 和 querySelectorAll()。

  • dart:crypto 类库从 SDK 中移除了 (详见 http://pub.dartlang.org/packages/crypto)。

  • AsyncError class 被移除了 (从 dart:async 中)。

  • dart:isolate 类库被重构了， 现在只有核心的 primitives. 书中我们移除了它的大部分介绍内容，但是我们期待更高水平的 API 能够开发出来使隔离更容易

  • main() 方法现在拥有一个可选的参数 (List<String>)。

讲解工具的章节也做了相应的调整：

  • dart_analyzer 工具被 dartanalyzer 取代了

  • 很多工具的命令行参数调整了。例如 dartdoc --pkg 命令行选项 变更为 --package-root。

  • pub 工具现在有 build 和 serve 命令集了, 它们可以让转换器将源代码文件转换成最终格式。

  • 我们完善了 dart2js 文档。

我们同样增加了以前无正式文件说明的特性的范围,并稍稍调整已有的范围。在语言的简介中：

  • 添加了 the section called “在类中添加功能： Mixins（混入）”。

  • 在 the section called “关键字” 中, 标记出哪些关键字是内建的标识符，并论述了他们与保留标识符的不同。

  • 增加了 + 操作符的功能，现在可以作用于字符串了。

  • 论述了使用表达式（例如 function calls）作为非默认构造函数参 数的相关问题。

  • 添加了一些关于 @override 和 @proxy 注释相关的例子。

  • 修改了文档注释的例子，使他们符合最新的规则,并且指向了 Dart 文档注释指南。

在类库的概览中：

  • 增加 the section called “dart:mirrors - Reflection（反射）”。

  • 添加关于解析非十进制数字的例子。

  • 在使用 Future.wait() 的例子中，错误使用 new 的部分移除了。

  • 移除完成符（Completer）的章节，它不再被推荐了。

  • 添加 the section called “Stream”。

我们更新了全书中所有相关主题的链接。

第二次印刷中的修改： 2013年3月29日

我们更新了相关内容以反应出以下第二次印刷中的修改：

  • 现在必须在附加类库文件中使用 part of ， 而不再是可选的了。

  • 在 M2 中，调整了一些 API ：

    • 在 dart:html 类库中, Element 的 elements 属性改为了 children。

    • dart:html 类库中的名字做了调整，现在更加符合 Dart 命名规范了。 特别的, innerHTML 修改为 innerHtml。

    • String 的charCodes() 方法修改成了一个 getter ，因此我们移除所有引用的圆括号，charCodes。

    • File 的 readAsText() 方法变成了 readAsString()。

    • Date 类的构造函数做了调整。

    • 删除了 NullPointerException 。

    • 删除了 RegExp 的常量构造函数

    • StringBuffer 的 add() 方法的返回值类型修改为 void。 现在连续调用 add() 的代码现在可以使用 级联方式了。

  • 现在使用dart.js推荐的方式是生成一个本地的副本，更好的方式是下载使用browser pub 包。

  • Dart 语言现在支持 Metadata 了。 请查看 the section called “Metadata（元数据）”。

  • 我们增加了Web UI 包 的参考，这个包为创建 web 应用的 UI 界面提供 了一个高级的，可升级的方法。

  • 在 M3 中，mixins 被添加到了语言中，可以重用不同类中的代码。

  • 在 M3 中核心库被很大程度的优化了。包括了一下的修改：

    • 一些方法被修改为域或者 getters。这意味着我们应该在hashCode,isEmpty, 和 isNan后面移除() 。我们同样将 getKeys() 改为 keys, 将 getValues() 改为 values。

    • Iterable 类得到了增强，所有的 Collections 都收到了影响，例如 Lists 和 Sets。以前 Collection 中的方法filter() 和 map() 移到了 Iterable 中，并且filter()的名字改为where(). (map() 暂时改为 mappedBy(),是由于社区反馈这个决定是对其是倒退发展。) some() 方法改为 any()。Iterables 返回的很多值现在是惰性填充Iterables；你可以使用toList() 或 toSet() 强制赋值。我们推荐继承 Iterable，而不是只是实现它，因此你可以从添加新功能中得益。更多的细节请查看文章 Iterables.

    • Iterator 接口的 next() 和 hasNext() 改为 current 和 moveNext()。

    • JSON 类不再放在 dart:json 库中了。以前的 JSON 静态方法如 parse() 和 stringify() 现在便趁个了最顶层功能。

    • Date 类更名为 DateTime, 并且 fromString() 构造函数现在变成了一个静态方法，命名为 parse()。

    • Event-handler 过去的注册形式是 .on.event.add(); 现在改为 .onEvent.listen。

    • dart:html 类库中 HttpRequest 类的 get() 方法 被 getString() 和 request() 取代了, 相应分别返回 Future<String> 和 Future<HttpRequest>。

    • dart:html Window 的setTimeout() 方法过时了; 对应的, 使用 Future.delayed() 或者如果你知道你正在做什么，使用一个 Timer。

    • Timer and Completer 从 dart:isolate 库中转移到一个新的名字叫做 dart:async 的库中。 Timer 构造函数现在使用一个 Duration 做参数，取代了 int.

    • Future类的 chain() 和 handleException() 方法过时了, 现在使用 then() 和 catchError()。

    • String 类的 splitChars() 方法过时了; 相应的, 使用一个空字符串参数的 split() 方法。 String 的charCodes getter 和 charCodeAt() 方法过时了; 如果要获取 UTF-16 code 单元, 现在使用 String 的 codeUnits 或者 codeUnitAt() 。

    • StringBuffer 类的add() 和 addAll() 方法分别被 write() 和 writeAll()取代了。

    • dart:io 库在 M3 后改变很大, 现在 Dart 所有的 I/O 功能都集中 在这个库中。 I/O 现在以 dart:async 库的 Stream 和 Future 类为核心, 取代了 callbacks。 InputStream 和 OutputStream 类被实现了 Stream 和 IOSink 的类所取代。 一个新的 FileSystemEntity 类是 File 和 Directory 的父类。 如果要创建一个 HttpServer, 你现在需要使用 bind() 方法。更多相关信息请查看 相关通告。

    • dart:crypto 库中许多关于 hashes 的方法都由 update() 和 digest() 改为了 add() 和 close()。

我们修正或者改写了一些内容，例如:

  • 按位操作符是用 int 实现的, 而不是 num 。

  • is 和 as 例子examples (在 the section called “类型测试操作符” 中) 并不是完全等价的。

我们添加了一些关于预先浏览的特性的内容：

  • 关键字 (请见 the section called “关键字”)

  • Lexical scope (请见 the section called “（语法作用域）Lexical Scope”)

  • Function equality (请见 the section called “判断函数相等”)

  • dart_analyzer 工具 (请见 the section called “dartanalyzer: 静态分析工具”)

  • dartdoc 工具 (请见 the section called “dartdoc: API 文档生成器”)

最后, Figure 1.1, “浏览器速度 (V8 benchmark suite v7; 数字越大性能越好)” 现在使用一个最新的基准，并且用最新的数字。
